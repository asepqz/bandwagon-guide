## 三条线路，先搞清楚再选套餐

买个搬瓦工养老，最容易卡住的地方不是价格，是线路。搬瓦工卖的套餐名字本身就带线路信息，没搞懂这个，买完发现跑的机房跟想的不一样，就难受了。

**CN2 GT（入门线路）**：国际段走 CN2 网络，到国内段回落到普通骨干。便宜，晚高峰容易堵，延迟波动大。现在搬瓦工的 KVM 套餐走的基本是这条或者更普通的线路，适合对速度没什么要求、纯粹想要一台便宜机器跑任务的人。

**香港 / 日本大阪 CN2 GIA（亚太直连）**：延迟低到很难再低。香港机房延迟大概 30ms 出头，大阪也就 50ms 上下。价格也对应上去了，不是日常选择，是"钱不是问题"的选择。

说实话，大多数人买搬瓦工养老，CN2 GIA-E 套餐就够了，日常用起来不会有什么挑剔的地方。

## 机房怎么选：三个使用场景的具体建议

问题是买完 CN2 GIA-E 套餐，默认机房是哪个？应该切换到哪里？

**场景一：普通国内用户，三网都有但以电信为主**

洛杉矶 DC6（USCA\_6）直接选。这是 CN2 GIA-E 的核心机房，电信双程 CN2 GIA，联通也走 CN2，移动回程 CMIN2。延迟在 150ms 上下，晚高峰基本稳。用这个机房放着不动，是最省心的选法。

**场景二：移动用户或者对延迟更敏感**

先切到日本软银机房（JPOS\_1）试试。软银对移动、联通用户很友好，延迟比洛杉矶低不少。用过的人通常提到它稳定性不错，就是月流量按照软银机房计算会有折扣，注意看一下。

**场景三：对延迟要求真的很高**

香港或者日本大阪 CN2 GIA 就是你要找的。延迟低，速度快，就是价格跨了一个台阶。大阪 CN2 GIA 是三网回程 CN2 GIA，综合下来性价比比香港更合适，但也贵过 CN2 GIA-E 一倍多。

顺便提一嘴：荷兰 EUNL\_9 机房走的是联通 9929 线路，如果你主要面向的是欧洲用户，或者联通接入为主，可以试一试这个机房，速度通常也不差。
